## 🎯 What is ProofHire?

ProofHire solves two critical problems in tech recruitment:

**Problem 1 — Recruiters are overwhelmed**
A recruiter at Umurava receives 200+ applications per job. Manual screening takes 3-4 days and introduces bias. ProofHire uses Gemini AI to screen all candidates simultaneously in under 30 seconds.

**Problem 2 — Profiles are unverified**
Anyone can write "Expert React Native" on their profile. No one checks. ProofHire introduces a "Proof of Skill" feature — a 5-minute AI-generated test that verifies claimed skills before the candidate reaches the recruiter.

**Our differentiator — Africa-First AI**
Existing AI screeners penalize African developers for not having FAANG experience or Western university degrees. ProofHire's Gemini prompt is specifically engineered to value African context: hackathon wins, local impact projects, and real-world contributions from Kigali, Lagos, and Nairobi are scored fairly.

## 🤖 AI Decision Flow

```
Step 1 ──► Recruiter creates a job with required skills and experience level

Step 2 ──► Candidates are ingested:
           Option A: Structured JSON profile (Umurava schema)
           Option B: CSV file upload (parsed automatically)

Step 3 ──► Recruiter clicks "Run AI Screening"
           └─► Backend fetches all applicants for that job
           └─► Compiles ALL profiles into ONE structured Gemini prompt
           └─► System instruction enforces Africa-first fairness rules:
               "Judge based on skills, not company names.
                A dev from Kigali with strong projects is as valuable
                as one with FAANG experience."

Step 4 ──► Gemini 1.5 Flash evaluates ALL candidates simultaneously
           └─► Returns a ranked JSON array with:
               - matchScore (0-100)
               - strengths (3-5 specific points)
               - gaps (2-4 constructive points)
               - recommendation (Hire / Maybe / Pass)
               - reasoning (2-3 sentence explanation)
               - rank (no ties allowed)

Step 5 ──► Results saved to MongoDB, displayed in recruiter dashboard
           └─► Top 10 shown by default, toggle to Top 20
           └─► Each card expandable to show full AI reasoning

Step 6 ──► Candidate takes Proof of Skill test:
           └─► 5 questions generated by Gemini based on declared skills
           └─► correctAnswer NEVER leaves the server (security)
           └─► 5-minute countdown timer (auto-submits on timeout)
           └─► 70%+ = isVerified: true in MongoDB = Verified badge

Step 7 ──► Recruiter makes the final hiring decision (human in control)
```

## 🛠️ Local Development Setup

### Prerequisites

- Node.js 18 or higher
- npm 9 or higher
- MongoDB Atlas account (free tier at mongodb.com/atlas)
- Google AI Studio API key (free at aistudio.google.com)

### Step 1 — Clone the repositories

```bash
git clone https://github.com/[USERNAME]/proofhire-backend
git clone https://github.com/[USERNAME]/proofhire-frontend
```

### Step 2 — Setup the Backend

```bash
cd proofhire-backend

# Install dependencies
npm install

# Create your environment file
cp .env.example .env

# Edit .env with your credentials:
# PORT=5000
# MONGODB_URI=mongodb+srv://user:password@cluster.mongodb.net/proofhire
# GEMINI_API_KEY=your_gemini_api_key_here
# FRONTEND_URL=http://localhost:3000
# NODE_ENV=development

# Seed the database with 15 test profiles and 3 jobs
npm run seed

# Start the development server
npm run dev
# ✅ Backend running on http://localhost:5000
```

### Step 3 — Setup the Frontend

```bash
cd proofhire-frontend

# Install dependencies
npm install

# Create your environment file
echo "NEXT_PUBLIC_API_URL=http://localhost:5000" > .env.local

# Start the development server
npm run dev
# ✅ Frontend running on http://localhost:3000
```

### Step 4 — Test the full flow

1. Open http://localhost:3000
2. Go to **Jobs** → Create a new job
3. Go to **Applicants** → Upload CSV or import a JSON profile
4. Go to **Screening** → Click "Run AI Screening"
5. View the ranked results with AI reasoning
6. Click "Take Skill Test" on any candidate
7. Complete the 5-minute test and see the Verified badge

## ⚠️ Assumptions & Limitations

- **No authentication in MVP**: The current version is open access. Authentication and role-based access control (recruiter vs. candidate roles) are planned for the production version.
- **Gemini rate limits**: Screening large batches (50+ candidates) may approach API rate limits. The current implementation uses a single prompt per screening session for maximum speed.
- **CSV import scope**: CSV upload supports basic profile fields (name, email, headline, location, skills). Full nested data (experience history, education, projects) requires JSON profile format.
- **Test integrity**: The Proof of Skill test is not proctored. Correct answers never leave the server, but screen sharing cannot be prevented in this MVP version.
- **MongoDB Atlas free tier**: The M0 free cluster supports up to 512MB storage and 100 connections — sufficient for hackathon demonstration and early production use.
- **AI decisions are advisory**: ProofHire's AI scores and recommendations are designed to assist recruiters, not replace human judgment. Final hiring decisions always remain with the recruiter.
